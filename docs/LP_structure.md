1. 에이전트 구성(권장)과 책임
   A. 전략/리서치 에이전트(Strategy)

* 타깃, 시장의 통념, 반대논리, 경쟁 포지션, “문제 재정의” 근거 수집
* 산출물: 1페이지 브리프(아래 ‘입력 시트’)

B. 카피 에이전트(Copy)

* 11개 블록 카피 작성(헤드라인~FAQ~CTA)
* 산출물: 카피 원고(블록별), CTA 문구 세트 10개

C. 증거/검증 에이전트(Proof)

* 숫자/사례/성과/후기/데이터의 출처 확인 및 “검증가능한 형태”로 변환
* 산출물: 증거팩(Evidence Pack)

D. 오퍼/상품구조 에이전트(Offer Architect)

* 제공물 구조, 가격/결제, 보너스, 리스크 리버설, 후속 경로 설계
* 산출물: 오퍼 시트(Offer Sheet)

E. UX/IA 에이전트(UX)

* 스크롤 흐름, 섹션 길이, CTA 배치, 스키머용 요약 박스, 모바일 최적화
* 산출물: 섹션 IA, 레이아웃 메모(“어디에 무엇을”)

2. 설득 블록 템플릿(11-Block)
   각 블록은 “목표/필수 요소/카피 슬롯/검수 기준”으로 작성합니다.
   카피 에이전트는 슬롯을 채우고, Proof 에이전트는 ‘증거 슬롯’만 책임지고 채웁니다.

BLOCK 1. Attention(후킹)
목표: 스크롤을 멈추게 한다(3초)
필수 요소: 정체성/약속/차별점 중 2개 이상
카피 슬롯:

* 헤드라인: “{타깃의 욕망}을 {방식/프레임}으로 {결과}”
* 서브: “{통념을 깨는 한 문장}”
  검수 기준:
* 타깃이 “이건 내 얘기”라고 느끼는 단어가 들어갔는가
* 결과가 추상(성공/대박)이 아니라 구체(매출/리드/전환/시간)인가

BLOCK 2. Ethos(권위/신뢰 스택)
목표: “이 사람이 말할 자격이 있나”를 10초 내 통과
필수 요소: 이력 1개 + 성과 수치 1개 + 3rd-party 1개(가능하면)
카피 슬롯:

* “{전환 서사 한 줄}: {이전 상태} → {현재 상태}”
* “{기관/경력}: {검증 가능 명칭}”
* “{성과 수치}: {기간}/{조건} 포함”
  검수 기준:
* 숫자는 기간/조건이 붙었는가(“언제, 무엇을 기준으로”)

BLOCK 3. Problem(현실 진단)
목표: “내 문제를 정확히 알고 있다” 확신
필수 요소: 실패 패턴(대다수) + 감정(좌절) + 공통 행동(헛수고)
카피 슬롯:

* “대부분 {타깃}은 {실패 결과}에서 멈춥니다.”
* “왜냐하면 {원인 가설} 때문입니다.”
  검수 기준:
* 개인 탓이 아니라 구조 탓으로 재정의했는가(죄책감↓ 실행의지↑)

BLOCK 4. Reframe(통념 파괴 → 새 기준 제시)
목표: 독자의 사고를 “원하던 기준”에서 “우리 기준”으로 이동
필수 요소: 기존 통념 1개 반박 + 반례/사례 1개
카피 슬롯:

* “{통념}은 착각입니다. 실제로는 {새 기준}이 핵심입니다.”
* “{작은 수치 사례}: {구독자/팔로워/예산 적어도} → {결과}”
  검수 기준:
* 반례는 ‘운’이 아니라 구조로 설명되는가

BLOCK 5. Mechanism(작동 원리: 병목/시스템)
목표: “어떻게 되는지”가 머리에 그려지게
필수 요소: 비유(1개) + 병목 개념 + 요소 리스트(3~7개, 너무 길면 접기)
카피 슬롯:

* 비유: “{비유}처럼, {가장 약한 요소}가 전체를 결정합니다.”
* 요소: “{요소1}/{요소2}/{요소3}…”
  검수 기준:
* 독자가 “내 병목이 뭔지” 자가진단하고 싶어지는가

BLOCK 6. Anti-hype(과장 거부로 신뢰 획득)
목표: 방어심 제거(“또 사기 아냐?” 차단)
필수 요소: 과장 문구 2~3개를 직접 부정 + 우리의 원칙 1줄
카피 슬롯:

* “{과장 약속} 같은 말은 하지 않습니다.”
* “우리는 {측정/훈련/관리/반복}만 합니다.”
  검수 기준:
* 부정만 하고 끝나지 말고 “대신 무엇을 하겠다”로 연결했는가

BLOCK 7. Proof(데이터/증거)
목표: “말이 아니라 증거”로 설득 닫기
필수 요소: 지표 2종(결과 지표 1 + 과정 지표 1) + 표/그래프/전후 비교
증거 슬롯:

* 결과 지표: {매출/리드/전환/조회/유지}
* 과정 지표: {출석률/과제수행/유지율/반복횟수}
  검수 기준:
* 표가 이미지라면, 텍스트로 3줄 요약이 같이 있는가(스키머 대응)

BLOCK 8. Offer(제안: “강의”가 아니라 “시스템 상품”)
목표: 구매 이유를 ‘지식’이 아니라 ‘변화 장치’로 고정
필수 요소: 구성(코어 3개) + 제공 방식 + 기대 결과(조건부)
오퍼 슬롯:

* “이건 {컨설팅/강의}가 아니라 {트레이닝/시스템}입니다.”
* 코어 구성 3종: {코어1}, {코어2}, {코어3}
  검수 기준:
* 코어 구성은 “행동 변화”를 만드는 장치인가(피드백/측정/반복/관리)

BLOCK 9. Qualification(필터링)
목표: 미스매치 제거 + 진짜 타깃의 결심 강화
필수 요소: “이런 분은 신청하지 마세요” 4~6개 + “이런 분이라면 적합” 3개
카피 슬롯:

* 비권장: {유형1}/{유형2}/{유형3}…
* 권장: {유형A}/{유형B}/{유형C}
  검수 기준:
* 비권장 항목이 ‘비난’이 아니라 ‘조건/현실’ 기반인가

BLOCK 10. Logistics/FAQ(불안 제거)
목표: 결제 직전 질문을 선제 차단
필수 요소: 가격/일정/진행 방식/변경 규칙/환불 또는 정책/장소(또는 온라인)
FAQ 슬롯(최소 6개):

* “비용은?”
* “시간이 없는데 가능한가?”
* “초보도 가능한가?”
* “오프라인/온라인?”
* “무엇을 준비해야?”
* “성과가 안 나면?”
  검수 기준:
* 규칙이 모호하지 않고 “조건/예외/절차”가 있는가

BLOCK 11. CTA 회수(반복)
목표: 설득이 완결되는 지점마다 행동 유도
필수 요소: CTA 버튼/문장 + 마이크로 카피(불안 완화) + 2차 CTA(옵션)
CTA 슬롯:

* 1차 CTA: “{행동}하기”
* 마이크로: “{리스크 낮추는 문장: 예) 대기만 해도 무료 제공}”
* 2차 CTA: “{리드마그넷} 받기”
  검수 기준:
* CTA는 ‘지금’ 해야 할 이유(마감/좌석/선발/혜택)가 있는가

* 성과2:
* 성과3:

E2. 과정 지표 3개

* 출석률:
* 과제 수행률:
* 유지율/재구매율:

E3. 대표 사례 3개(짧은 카드)

* 사례A: {전} → {후} / {기간} / {무엇을 바꿨나(병목)}
* 사례B:

* “모든 사람이 되는 건 아님”을 보여주는 데이터 1개
* 탈락/중도 포기 조건(정직하게)

4. 품질 체크리스트(최종 검수)
   아래 10개 중 8개 미만이면 재작업 권장

* 헤드라인만 봐도 타깃/결과/차별점이 보인다
* 통념을 깨는 ‘리프레임’이 한 번 이상 들어간다
* 메커니즘이 비유+병목 형태로 설명된다
* 과장 거부(anti-hype)가 들어가 방어심이 내려간다
* 증거는 결과+과정 2종 지표를 가진다
* 오퍼는 “지식”이 아니라 “변화 장치”로 구성된다
* 필터링이 있어 미스매치 고객을 미리 걸러낸다
* FAQ가 정책 수준으로 명확하다(조건/절차)
* CTA가 최소 3회 등장한다(상단/중단/하단)
* 모바일에서 스키머용 요약(3줄 박스)이 있다

5. 에이전트용 작성 프롬프트(바로 복붙해서 쓰는 운영 문장)
   아래 문장을 각 에이전트에게 그대로 주면 됩니다.

[Strategy Agent Prompt]
“입력 시트를 채우기 위해 타깃의 통념/실패 패턴/반대 논리/차별 포지션을 정리하고, Problem→Reframe→Mechanism의 논리 연결을 1페이지 브리프로 작성하라. ‘구조 탓’으로 문제를 재정의하라.”

[Copy Agent Prompt]
“11-Block 템플릿에 맞춰 랜딩 카피를 작성하라. 각 블록은 120~220자 범위로 우선 작성하고, Proof가 채워지면 수치 문장을 교정하라. CTA는 10개 변주를 만들고, 마이크로 카피로 불안을 낮춰라.”

[Proof Agent Prompt]
“카피에 등장하는 모든 수치/사례/권위를 Evidence Pack 형식으로 정리하라. 기간/조건/증빙 형태가 없으면 ‘미검증’으로 표시하고 카피에서 제외 또는 약화 표현으로 바꿔라.”

[Offer Architect Prompt]
“오퍼를 ‘트레이닝 시스템’으로 설계하라. 코어 구성 3개(측정/피드백/반복) 중심으로 재구성하고, 리드마그넷(2차 CTA)으로 진입장벽을 낮춰라. 필터링 조건을 현실 기반으로 작성하라.”

[UX Agent Prompt]
“스크롤 흐름에서 이탈 지점을 줄이도록, 상단 3줄 요약 박스, CTA 위치(상/중/하), 모바일 접힘 처리(요소 리스트, 증거 표)를 설계하라.”

6. 실전 적용 방법(가장 빠른 운영 순서)
7. Strategy가 입력 시트 + 브리프 완성
8. Offer Architect가 오퍼 시트 확정(가격/구성/정책 포함)
9. Proof가 Evidence Pack 확정(카피에 넣을 수치만 선별)
10. Copy가 11-Block 1차 작성 → Proof 반영 2차 교정
11. UX가 IA/CTA 배치 확정 → 최종 합본
