# ABEL AI 랜딩페이지 설득 구조 v2

> **목적**: PREP 프레임워크 기반 설득 구조 재설계
> **타깃 독자**: SEO/AEO/GEO 개념은 모르지만, AI 답변에 나오는 것의 중요성은 인지하는 중소기업 의사결정자
> **톤**: 시니어 컨설턴트 — 자신감 있지만 강압적이지 않음. 데이터로 말함
> **금지**: AI 느낌 문구, 과장 수식어, 강압적 긴급성 ("지금 당장!", "놓치지 마세요!")
> **작성일**: 2026-02-11

## 전체 설득 프레임워크: PREP 매크로 구조

페이지 전체가 하나의 PREP 논증을 형성합니다.

| PREP | 섹션 | 역할 | 고객 심리 |
|------|------|------|----------|
| **P**oint | Hero | "AI 답변에 당신이 없습니다" | "맞아, 우리도 안 나오더라" |
| **R**eason | Reality | "검색 환경이 바뀌었기 때문입니다" | "그래서 그랬구나" |
| **E**xample | Insight → Services → Process → Differentiators | 해결 방법과 증거 | "이렇게 하면 되는구나" |
| **P**oint | FAQ → Final CTA | 의심 해소 → 행동 촉발 | "한번 알아보자" |

### 핵심 원칙

1. **Open Loop**: 각 섹션 끝에 다음 섹션을 보고 싶게 만드는 질문/궁금증 유발
2. **고객 언어**: AEO/GEO/SEO 용어는 Insight 섹션(3번째)에서 처음 등장. 이전에는 고객이 느끼는 현실로 서술
3. **Feature → Benefit**: 모든 서술의 주어는 "우리"가 아닌 "당신"
4. **데이터 앵커**: 주장에는 반드시 수치 근거 (SEO_STRATEGY.md 기반)

## 섹션 1: Hero — "문제 인식"

### 역할
즉각적 공감 + 스크롤 유도. 방문자가 3초 안에 "이건 나 얘기다"라고 느끼게 만듦.

### 기법
공감적 질문 — 고객이 이미 느끼고 있지만 명확히 표현하지 못한 불안을 거울처럼 비춤.

### 카피 초안

**제목**:
```
고객이 AI에게 물었습니다.
당신은 답변에 없었습니다.
```

**부제**:
```
ChatGPT에 업종을 검색하면, 경쟁사만 나옵니다.
답변에 없으면, 선택지에도 없습니다.
```

**CTA**: `무료 진단 요청` (즉시 행동할 준비가 된 사람용, 상단 1개만)

### 전환 훅 (→ Reality)
```
왜 이런 일이 벌어지고 있을까요?
```
> 시각적으로 부드러운 스크롤 유도. 텍스트 또는 미세한 화살표 애니메이션.

### 현재 대비 변경점
| 현재 | 변경 | 이유 |
|------|------|------|
| 부제에 "AEO·GEO·SEO" 등장 | 전문 용어 완전 제거 | 타깃 독자는 이 용어를 모름 |
| "ABEL AI가 ~를 만듭니다" (자사 중심) | 고객의 현실을 비추는 구조 | 공감 우선 |
| "ChatGPT 월간 사용자수 2000만+명" (수치 나열) | Reality 섹션으로 이동 | Hero는 감정, 수치는 그다음 |

### 톤 가이드
- 담담하게 팩트를 짚는 시니어 컨설턴트
- 공포 조장 아님 — "당신의 현실입니다"라는 직시
- 마침표로 끝나는 짧은 문장. 느낌표 금지

## 섹션 2: Reality — "시장 변화"

### 역할
문제의 구조적 원인 설명. "내 탓이 아니라 시장이 바뀐 것"이라는 프레이밍으로 방어심 해제 + 긴급성 부여.

### 기법
PAS의 Agitate — 데이터로 뒷받침. 공포가 아닌 현실 직시.

### 카피 초안

**제목**:
```
고객이 정보를 찾는 방식이 달라졌습니다
```

**부제**:
```
검색창에 입력하던 고객이, 이제 AI에게 묻습니다.
```

**본문 (데이터 카드 3개)**:

| 수치 | 설명 |
|------|------|
| **54.5%** | 한국인 절반 이상이 ChatGPT로 정보를 검색합니다 |
| **69%** | 검색 결과를 클릭하지 않고 AI 답변만 보고 떠납니다 |
| **80.92%↑** | AI 검색 트래픽, 전년 대비 급증하고 있습니다 |

**보조 문장**:
```
검색 1등이어도, AI 답변 밖이면 고객은 당신을 보지 못합니다.
이것은 트렌드가 아닙니다. 이미 일어나고 있는 일입니다.
```

### 전환 훅 (→ Insight)
```
그렇다면, 답변에 포함되려면 무엇이 필요할까요?
```

### 현재 대비 변경점
| 현재 (ValueStatement) | 변경 (Reality) | 이유 |
|----------------------|----------------|------|
| "답변을 점령하면 시장을 점령합니다" (1줄) | 데이터 카드 3개 + 맥락 설명 | ValueStatement는 너무 짧아 설득력 부족 |
| 수치 근거 없음 | SEO_STRATEGY.md의 핵심 수치 활용 | 데이터가 긴급성을 만듦 |
| 선언적 톤 | 현실 직시 프레이밍 | "당신 탓이 아니라 세상이 바뀐 것" |

### 데이터 출처
- 한국인 54.5% ChatGPT 사용: 2026 OpenSurvey
- 제로클릭 69%: SEO_STRATEGY.md
- AI 검색 트래픽 80.92%↑: Conductor 2026

## 섹션 3: Insight — "핵심 인사이트" (신규)

### 역할
"aha moment" — 문제를 기회로 전환. "해결할 수 있구나"라는 깨달음. 여기서 처음으로 AEO/GEO/SEO 개념을 고객 언어로 번역하여 도입.

### 기법
Reframe (문제 → 기회). 전문 용어를 고객이 이해할 수 있는 결과 언어로 치환.

### 카피 초안

**제목**:
```
답변에 나오는 것은 운이 아닙니다
```

**부제**:
```
AI가 특정 브랜드를 답변에 포함하는 데는 구조가 있습니다.
그 구조를 설계하는 것이 우리가 하는 일입니다.
```

**본문 (3개 인사이트 카드)**:

각 카드는 "고객이 얻는 결과 → 이를 위한 접근 → 전문 용어(작게)" 순서.

**카드 1: 직접 답변**
```
"OO 추천해줘"라고 물었을 때, AI가 당신을 먼저 말하게 합니다.

질문에 바로 답할 수 있는 콘텐츠 구조를 설계합니다.
AI 엔진은 명확한 답변이 있는 콘텐츠를 선호합니다.

→ Answer Engine Optimization (AEO)
```

**카드 2: 출처 인용**
```
AI가 답변할 때, 당신의 사이트를 출처로 링크하게 합니다.

근거와 데이터가 있는 신뢰 구조를 만들어,
AI가 "참고한 출처"로 당신을 인용하도록 설계합니다.

→ Generative Engine Optimization (GEO)
```

**카드 3: 검색 전환**
```
검색에서 발견되고, 발견이 문의로 이어지게 합니다.

유입 키워드부터 전환 동선까지,
검색 → 방문 → 문의의 전체 여정을 최적화합니다.

→ Search Engine Optimization (SEO)
```

**통합 메시지**:
```
이 세 가지는 따로 작동하지 않습니다.
답변 + 출처 + 검색을 하나로 설계해야 결과가 나옵니다.
```

### 전환 훅 (→ Services)
```
구체적으로 어떤 서비스인지 살펴보세요.
```

### 현재 대비 변경점
- **완전 신규 섹션**. 현재 구조에는 전문 용어를 고객 언어로 번역하는 구간이 없음
- 기존에는 Services 섹션에서 AEO/GEO/SEO를 기술 설명으로 직접 제시 → 이해도 낮음
- Insight가 "번역기" 역할을 하여, Services에서 전문 용어가 나와도 이미 맥락을 이해한 상태

## 섹션 4: Services — "솔루션"

### 역할
구체적 서비스 제시. Insight에서 형성된 이해를 바탕으로, 각 서비스가 고객에게 가져다주는 결과를 리드.

### 기법
Feature → Benefit 변환. 모든 카드의 첫 문장은 "고객이 얻는 것".

### 카피 초안

**제목**:
```
답변에 나오는 구조를 만듭니다
```

**3개 서비스 카드**:

---

**AEO — AI 답변의 첫 번째 선택지가 됩니다**

> AI가 당신의 콘텐츠를 직접 답변으로 채택합니다.

- 질문형 의도 분석 및 답변형 카피 설계
- 답변 우선 구조 (정의 → 요약 → 근거) 구성
- FAQ/피처 스니펫 대응 콘텐츠 제작

---

**GEO — AI가 당신을 출처로 인용합니다**

> 답변 하단의 "참고한 사이트"에 당신의 링크가 걸립니다.

- 인용 가능한 신뢰 구조 설계 (근거·출처·데이터)
- 브랜드/서비스 지식 체계화
- 생성형 검색 친화형 콘텐츠 정렬

---

**SEO — 검색 유입이 실제 문의로 이어집니다**

> 검색에서 발견되고, 발견이 전환으로 연결됩니다.

- 유입 키워드 구조 재설계
- 전환 동선/페이지 흐름 최적화
- 기술 SEO 기본 정비 (속도/인덱싱/메타)

---

### 전환 훅 (→ Process)
```
진행은 단순합니다.
```

### 현재 대비 변경점
| 현재 | 변경 | 이유 |
|------|------|------|
| oneLiner가 기술 설명 중심 | 고객이 얻는 결과가 첫 문장 | Feature → Benefit |
| subtitle가 영문 풀네임 | 풀네임은 Insight에서 이미 소개됨 → 여기서는 결과 강조 | 중복 제거, 결과 중심 |
| deliverables 항목 있음 | 삭제 (프로세스 섹션에서 "받는 것"으로 통합) | 정보 과부하 방지 |

## 섹션 5: Process — "진행 방식"

### 역할
진입 장벽 최소화. "이렇게 간단한데 안 해볼 이유가 없다"는 인식 형성.

### 기법
단순화 — 3단계로 압축. 각 단계에서 고객이 "받는 것"을 명시.

### 카피 초안

**제목**:
```
3단계. 그것으로 충분합니다.
```

**부제**:
```
복잡한 준비는 필요 없습니다. 진단 요청 하나로 시작됩니다.
```

---

**Step 1 — 진단**
```
AI 답변에서 당신의 현재 위치를 확인합니다.
```
당신이 받는 것: **AI 노출 현황 리포트** (무료)
- ChatGPT, Perplexity, Google AI Overview에서의 브랜드 노출 상태
- 경쟁사 대비 포지션 분석
- 콘텐츠 답변 적합성 평가

---

**Step 2 — 설계**
```
데이터 기반 통합 전략서를 받습니다.
```
당신이 받는 것: **AEO·GEO·SEO 통합 전략서**
- 타깃 키워드 및 질문형 쿼리 맵
- 콘텐츠 구조 설계안
- 전환 동선 개선안

---

**Step 3 — 실행**
```
콘텐츠와 구조가 최적화되고, 주간으로 관리됩니다.
```
당신이 받는 것: **실행 + 주간 운영**
- 답변형 콘텐츠 제작 및 구조 최적화
- 주간 업데이트 및 성과 리포트
- 답변 노출 빈도·인용 빈도·전환율 추적

---

### 전환 훅 (→ Differentiators)
```
왜 ABEL AI인가?
```

### 현재 대비 변경점
| 현재 | 변경 | 이유 |
|------|------|------|
| 각 단계 1줄 설명 | "당신이 받는 것" + 구체적 산출물 | 고객은 "뭘 받는지"가 궁금함 |
| 무료 진단 강조 약함 | Step 1에 "(무료)" 명시 | 진입 장벽 제거 |
| 기술 용어로 설명 | 결과 언어로 전환 | 고객 관점 서술 |

## 섹션 6: Differentiators — "왜 ABEL인가"

### 역할
경쟁사 대비 선택 이유 제시. "다른 곳이 아니라 여기여야 하는 이유".

### 기법
대조법 — 기존 방식과 ABEL의 접근을 명확히 비교.

### 카피 초안

**제목**:
```
기존 방식과는 다릅니다
```

**부제**:
```
검색 순위만 올리는 시대는 끝났습니다.
```

**대조 테이블**:

| 기존 방식 | ABEL AI |
|----------|---------|
| 검색 순위 올리기에 집중 | 답변 노출 + 출처 인용 + 검색 순위를 동시에 설계 |
| 한번 세팅하고 끝 | 주간 업데이트로 지속 관리 |
| 감과 경험에 의존 | 데이터와 근거 기반으로 설계 |
| 구글 검색만 대응 | 구글 + ChatGPT + 네이버 AI를 동시에 대응 |

**보조 문장**:
```
AI 엔진은 최신 정보를 선호합니다.
한번 세팅하고 방치하면, 답변에서 밀려납니다.
ABEL AI는 주간 단위로 콘텐츠를 갱신하고, 성과를 추적합니다.
```

### 전환 훅 (→ FAQ)
```
궁금한 점이 있으신가요?
```

### 현재 대비 변경점
| 현재 | 변경 | 이유 |
|------|------|------|
| 4개 독립 카드 (자사 설명) | 대조 테이블 (기존 vs ABEL) | 비교하면 차이가 선명해짐 |
| "우리가 하는 것" 중심 | "기존 방식은 이렇고, 우리는 다르다" | 고객은 비교로 판단함 |
| Process 전에 위치 | Process 후에 위치 | 진행 방식을 본 후 "왜 여기인가" 질문 자연스러움 |

## 섹션 7: FAQ — "의심 해소"

### 역할
남은 저항감 제거. CTA 직전에 위치시켜, 마지막 의심을 해소한 뒤 바로 행동으로 연결.

### 기법
Objection handling — 교육 → 실무 → 행동 순서로 재배치.

### FAQ 순서 재배치

기존 12개 FAQ를 3개 그룹으로 재배치합니다. 내용은 이미 잘 구성되어 있으므로 그대로 유지.

**그룹 1: 교육 (개념 이해)**
1. AI 검색 최적화가 왜 중요한가요?
2. AEO와 GEO는 무엇이 다른가요?
3. SEO만 하면 충분하지 않나요?
4. ChatGPT에 우리 브랜드가 노출되게 하려면?

**그룹 2: 실무 (서비스 이해)**
5. ABEL AI는 어떤 회사인가요?
6. 기존 SEO 대행사와 무엇이 다른가요?
7. 어떤 업종이 효과가 큽니까?
8. 콘텐츠 업데이트는 얼마나 자주 하나요?
9. 성과는 어떻게 측정하나요?

**그룹 3: 행동 (의사결정)**
10. 진행 기간은 얼마나 걸리나요?
11. 비용은 어떻게 책정되나요?
12. 무료 진단에서는 무엇을 확인하나요? ← **마지막 FAQ**

### 전환 훅 (→ Final CTA)
마지막 FAQ("무료 진단에서는 무엇을 확인하나요?")의 답변이 진단 내용을 구체적으로 설명하므로, 자연스럽게 "그러면 한번 받아보자"로 연결.

> FAQ 섹션과 Final CTA 사이에 별도 전환 문구 불필요. 마지막 FAQ 자체가 CTA 프라이밍.

### 현재 대비 변경점
| 현재 | 변경 | 이유 |
|------|------|------|
| 순서가 임의적 | 교육 → 실무 → 행동 순 | 이해 → 신뢰 → 결정의 심리 흐름 |
| 페이지 맨 끝 | CTA 직전 | 의심 해소 직후 행동 유도가 전환율 높음 |
| 회사 소개가 첫 번째 | AI 검색 중요성이 첫 번째 | 고객 관심사 우선 |

## 섹션 8: Final CTA — "행동 촉발"

### 역할
전환. 페이지의 모든 설득이 수렴하는 지점.

### 기법
Risk reversal + 구체적 혜택 명시. 고객이 잃을 것이 없다는 확신.

### 카피 초안

**제목**:
```
5분이면 시작할 수 있습니다
```

**부제**:
```
무료 진단을 요청하시면,
AI 답변에서 당신의 현재 위치와 개선 방향을 확인할 수 있습니다.
```

**받는 것 명시 (체크리스트)**:
```
- ChatGPT / Perplexity / Google AI에서의 현재 노출 상태
- 경쟁사 대비 포지션
- 콘텐츠 답변 적합성 평가
- 구체적 개선 방향 제시
```

**리스크 제거**:
```
무료. 약정 없음.
```

**CTA 버튼**: `무료 진단 요청`

### 현재 대비 변경점
| 현재 | 변경 | 이유 |
|------|------|------|
| "지금, 무료 진단을 요청하세요" (명령형) | "5분이면 시작할 수 있습니다" (용이성) | 행동의 가벼움 강조 |
| "현재 노출 상태와 개선 방향" (추상적) | 4개 체크리스트로 구체화 | 고객은 "뭘 받는지" 명확할 때 행동함 |
| 리스크 제거 문구 없음 | "무료. 약정 없음." 명시 | 마지막 저항 제거 |

## 전체 구조 비교표

### 섹션 순서 비교

| # | 현재 구조 | 새 구조 | 변경 이유 |
|---|----------|--------|----------|
| 1 | Hero (기술 용어 포함) | Hero (고객 언어) | 공감 먼저, 용어는 나중 |
| 2 | ValueStatement (1줄) | Reality (데이터 기반) | 수치로 긴급성 뒷받침 |
| 3 | Services (기술 설명) | Insight (신규) | "aha moment" 제공, 용어 번역 |
| 4 | Differentiators (방법론) | Services (Benefit 중심) | 고객이 얻는 것 리드 |
| 5 | Process (1줄 설명) | Process (고객 관점) | "받는 것" 중심 서술 |
| 6 | FAQ (12개, 임의 순서) | Differentiators (대조법) | 비교로 차별점 선명화 |
| 7 | FinalCTA (Hero 반복) | FAQ (재배치) | 의심 해소 → 전환 직전 |
| 8 | — | FinalCTA (구체적) | 받는 것 명시 + 리스크 제거 |

### 설득 흐름 비교

**현재**: 문제 제시 → (약한 연결) → 기술 설명 → 차별점 → 프로세스 → FAQ → CTA

**새 구조**: 공감(Hero) → 왜?(Reality) → 해결 가능(Insight) → 어떻게?(Services) → 얼마나 쉬운가?(Process) → 왜 여기?(Differentiators) → 남은 의심 해소(FAQ) → 행동(CTA)

### Open Loop 체인

```
Hero: "왜 이런 일이 벌어지고 있을까요?"
  ↓
Reality: "그렇다면, 답변에 포함되려면 무엇이 필요할까요?"
  ↓
Insight: "구체적으로 어떤 서비스인지 살펴보세요."
  ↓
Services: "진행은 단순합니다."
  ↓
Process: "왜 ABEL AI인가?"
  ↓
Differentiators: "궁금한 점이 있으신가요?"
  ↓
FAQ: (마지막 FAQ가 무료 진단 내용 → 자연스러운 CTA 연결)
  ↓
Final CTA: "5분이면 시작할 수 있습니다"
```
